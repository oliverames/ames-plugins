# ames-claude

Oliver's personal Claude Code plugins, skills, and marketplace ‚Äî all in one repo.

## Structure

```
plugins/<name>/                          ‚Äî Claude Code plugins (each has .claude-plugin/plugin.json)
plugins/ames-standalone-skills/skills/   ‚Äî Standalone skills (edit here directly, no build step)
.claude-plugin/marketplace.json          ‚Äî Marketplace manifest (auto-generated by ./sync)
sync                                     ‚Äî Regenerates marketplace.json from plugin manifests
bump-and-sync                            ‚Äî Bumps ames-ynab version + runs sync + pushes
```

## Workflow

1. **Develop** ‚Äî edit plugins and skills in this repo
2. **Sync** ‚Äî run `./sync` to regenerate `marketplace.json`
3. **Commit and push** ‚Äî Claude Code polls the manifest and auto-updates installed plugins

`marketplace.json` is auto-generated by `./sync` ‚Äî never edit it directly.

## Plugins

MCP = has `.mcp.json` (registers MCP servers when plugin is enabled).

## Plugin Conventions

- Every plugin needs `.claude-plugin/plugin.json` with `name`, `version`, `description`, `author`
- `version` is required ‚Äî bump it whenever content changes so Claude Code detects an update
- MCP plugins declare servers in a flat `.mcp.json` at the plugin root (no outer `mcpServers` key, no `"type": "stdio"`)
- Use `${ENV_VAR}` in `.mcp.json` for secrets ‚Äî resolved from Claude Code `settings.json` env at runtime
- Skills go in `skills/<name>/SKILL.md` within the plugin directory
- Commands go in `commands/<name>.md` with frontmatter (name, description, allowed-tools)

## Skill Conventions (ames-standalone-skills)

- Each skill is a folder with a `SKILL.md` inside `plugins/ames-standalone-skills/skills/`
- SKILL.md frontmatter: `name` and `description` are required
- `./sync` auto-discovers all skill folders and lists them in `marketplace.json`

## Version Bumping

- **Patch** (1.0.0 ‚Üí 1.0.1): content tweaks, bug fixes
- **Minor** (1.0.0 ‚Üí 1.1.0): new skills, commands, or MCP servers added
- **Major** (1.0.0 ‚Üí 2.0.0): breaking changes or full rewrites

`bump-and-sync` handles version bumping + sync + push for `ames-ynab` automatically (called by the npm postpublish hook in the ynab-mcp-server repo).

## Marketplace

Registered in Claude Code as `ames-claude` pointing to `oliverames/ames-claude`.

After any change: run `./sync`, commit, push. Claude Code will detect the update on next marketplace refresh.

## Install Paths

| Component | Location |
|-----------|----------|
| Plugin cache | `~/.claude/plugins/cache/ames-claude/<name>/<version>/` |
| Marketplace cache | `~/.claude/plugins/marketplaces/ames-claude/` |
| Plugin registry | `~/.claude/plugins/installed_plugins.json` |

## Gotchas

- **Plugin name must not equal marketplace name** (`ames-claude`) ‚Äî causes install collision
- **`marketplace.json` is auto-generated** ‚Äî run `./sync`, never edit manually
- **MCP connector secrets**: plugin-spawned MCP servers don't inherit the `op run` environment. Secrets must be declared in `.mcp.json` via `${ENV_VAR}` (resolved from `settings.json` env block), or loaded from a plugin-local `.env` file by the server itself
- **Flat `.mcp.json` format**: no outer `mcpServers` wrapper, no `"type": "stdio"` ‚Äî matches the official plugin format

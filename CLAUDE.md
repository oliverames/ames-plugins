# ames-claude

Oliver's personal Claude Code plugins, skills, and marketplace тАФ all in one repo.

## Structure

```
plugins/<name>/                          тАФ Claude Code plugins (each has .claude-plugin/plugin.json)
plugins/ames-standalone-skills/skills/   тАФ Standalone skills (edit here directly, no build step)
.claude-plugin/marketplace.json          тАФ Marketplace manifest (auto-generated by ./sync)
sync                                     тАФ Regenerates marketplace.json from plugin manifests
bump-and-sync                            тАФ Bumps ames-ynab version + runs sync + pushes
```

## Workflow

1. **Develop** тАФ edit plugins and skills in this repo
2. **Sync** тАФ run `./sync` to regenerate `marketplace.json`
3. **Commit and push** тАФ Claude Code polls the manifest and auto-updates installed plugins

`marketplace.json` is auto-generated by `./sync` тАФ never edit it directly.

## Plugin Conventions

- Every plugin needs `.claude-plugin/plugin.json` with `name`, `version`, `description`, `author`
- `version` is required тАФ bump it whenever content changes so Claude Code detects an update
- MCP plugins declare servers in a flat `.mcp.json` at the plugin root (no outer `mcpServers` key, no `"type": "stdio"`)
- Use `${ENV_VAR}` in `.mcp.json` for secrets тАФ resolved from Claude Code `settings.json` env at runtime
- Skills go in `skills/<name>/SKILL.md` within the plugin directory
- Commands go in `commands/<name>.md` with frontmatter (name, description, allowed-tools)

## Skill Conventions (ames-standalone-skills)

- Each skill is a folder with a `SKILL.md` inside `plugins/ames-standalone-skills/skills/`
- SKILL.md frontmatter: `name` and `description` are required
- **`name` MUST be kebab-case matching the directory name** (e.g. `name: apple-notes-formatting` for `skills/apple-notes-formatting/`). Display-name style values like `"Apple Notes Formatting"` cause Cowork's marketplace validator to reject the entire plugin with a generic "Failed to update marketplace" error тАФ debugged at length 2026-04-11, see memory `ref_claude_skill_md_schema.md`.
- `./sync` regenerates `marketplace.json` from plugin.json files. Skills are auto-discovered by Claude Code at install time from the `skills/` directory тАФ do NOT list them explicitly in marketplace.json (causes path resolution issues).
- Every subdirectory of `skills/` MUST contain a `SKILL.md`. Dev workspaces without a SKILL.md (eval outputs, iteration data) belong elsewhere тАФ Cowork's validator trips on them.

## Version Bumping

- **Patch** (1.0.0 тЖТ 1.0.1): content tweaks, bug fixes
- **Minor** (1.0.0 тЖТ 1.1.0): new skills, commands, or MCP servers added
- **Major** (1.0.0 тЖТ 2.0.0): breaking changes or full rewrites

`bump-and-sync` handles version bumping + sync + push for `ames-ynab` automatically (called by the npm postpublish hook in the ynab-mcp-server repo).

## Marketplace

Registered in Claude Code as `ames-claude` pointing to `oliverames/ames-claude`.

After any change: run `./sync`, commit, push. Claude Code will detect the update on next marketplace refresh.

## Gotchas

- **Plugin name must not equal marketplace name** (`ames-claude`) тАФ causes install collision
- **`marketplace.json` is auto-generated** тАФ run `./sync`, never edit manually
- **MCP connector secrets**: plugin-spawned MCP servers don't inherit the `op run` environment. Secrets must be declared in `.mcp.json` via `${ENV_VAR}` (resolved from `settings.json` env block), or loaded from a plugin-local `.env` file by the server itself
- **Flat `.mcp.json` format**: no outer `mcpServers` wrapper, no `"type": "stdio"` тАФ matches the official plugin format
